# 硬件配置
- **舵机**：360° 连续旋转舵机（只能顺时针旋转，不可逆时针）
- **药槽**：共 8 个，编号 0~7，均匀分布在转盘上
- **微动开关 0（SW0）**：仅药槽 0（Logo 位/归零位/起始位）能触发
- **微动开关 1（SW1）**：药槽 1~7 经过时触发，用于计数和定位
- **触发逻辑**：转盘顺时针转动时，药槽边缘会按压微动开关产生触发信号

# 核心逻辑规则

## 1. 药槽与开关的对应关系
| 药槽编号 | 可触发开关 | 特殊说明 |
|---------|-----------|---------|
| 0       | SW0 专属  | Logo位、归零位、开机起始位 |
| 1~7     | SW1 专属  | 每经过一个药槽，SW1 触发一次 |

## 2. 开机初始化流程（寻零）
- 上电后，舵机以慢速顺时针转动
- 持续检测 SW0 状态
- 一旦 SW0 被触发（检测到药槽 0），立即停止舵机
- 将当前位置标记为 **Slot 0**
- **关键**：SW0 触发的同时，必须将 SW1 的内部计数器清零（`counter = 0`）
- 初始化完成，进入待机状态

## 3. 日常定位逻辑（转到指定药槽 N）
- 目标：从当前位置顺时针转到药槽 N（N ∈ [0,7]）
- 方法：启动舵机顺时针转动，同时监听 SW1 触发次数
- 每检测到一次 SW1 触发，计数器 +1
- 当计数器值等于目标药槽编号 N 时，停止舵机
- **注意**：药槽 0 只能通过 SW0 定位，不能通过 SW1 计数到达

## 4. 回到起始位（药槽 0）
- 任何时候需要回到 Slot 0：
  - 启动舵机顺时针转动
  - 持续检测 SW0
  - SW0 触发后立即停止
  - 计数器清零（`counter = 0`）

## 5. 状态管理
维护以下状态变量：
- `current_slot`：当前所在药槽编号（0~7）
- `target_slot`：目标药槽编号
- `sw1_counter`：SW1 触发计数（仅在 SW0 触发时清零）
- `is_homing`：是否正在寻零中
- `is_moving`：是否正在转动中

# 边界情况与容错处理（必须实现）

1. **防抖动**：微动开关触发需做 20~50ms 硬件消抖，避免一次按压产生多次计数
2. **超时保护**：寻零或转槽过程若超过 10 秒未完成，强制停止并报错
3. **SW1 误触发**：如果在非转槽状态下 SW1 意外触发，忽略该信号，不更新计数
4. **跨圈处理**：由于只能顺时针，从 Slot 5 到 Slot 3 需要经过 6→7→0→1→2→3，共 6 次 SW1 触发（经过 Slot 0 时由 SW0 处理，不计入 SW1）
5. **位置记忆**：断电后无记忆，每次开机必须重新寻零
