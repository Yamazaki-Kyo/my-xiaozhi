## 功能设计

### 一、语音交互（XiaoZhi 原生）

> 📄 相关文档：[微信通知 (MCP 工具注册)](wechat-push.md) · [舵机控制](hardware.md)

- **唤醒词**："你好小智"
- **对话能力**：通过云端 LLM 实现自然语言交互，可询问用药时间、药品说明、健康建议
- **语音指令**（MCP 触发）：
  - "定时开灯" / "几点关灯" → MCP 触发添加定时调度事件（LED 开关）
  - "推送聊天记录" → MCP 触发 WxPusher 微信推送当前对话摘要
  - "今天还有什么药没吃" → 查询今日未完成提醒

### 二、人脸识别（K230D 本地 NPU）

> 📄 相关文档：[K230D 视觉识别](k230d-vision.md) · [微信通知](wechat-push.md)

| 功能               | 状态          | 实现方式                                              | 应用                                 |
|--------------------|:-------------:|-------------------------------------------------------|--------------------------------------|
| **人脸识别**       | ✅ 当前实现   | MobileFaceNet 轻量模型，本地注册 2-5 位家庭成员       | 非注册人员取药微信告警；个性化提醒   |
| **药品识别**       | 🔲 规划中     | 自训练 YOLO 小模型，.kmodel 部署                      | 取药时视觉确认是否正确               |
| **条码扫描**       | 🔲 规划中     | K230D 内置二维码/AprilTag 库                          | 扫描药品包装条码自动录入             |
| **取药动作检测**   | 🔲 规划中     | 帧差法监控取药口区域                                  | 自动确认取药完成                     |

### 三、服药调度（XiaoZhi 本地 MedicineConfigServer）

> 📄 相关文档：[数据存储](data-and-screen.md) · [舵机控制](hardware.md) · [微信通知](wechat-push.md)

- **计划管理**：Web 页面 `/medicine-config` 配置日历事件 + 每日用药计划，7 槽独立药品（槽0为logo展示位，不放药）
- **数据持久化**：服药计划存储在 NVS，断电不丢失
- **RTC 定时触发**：30 秒周期检查（MedCheckCallback），匹配当前时间触发提醒
- **语音播报**：到点自动 OGG 语音循环播放 "该吃药了"（5.5 秒间隔循环），直到用户确认
- **自动转盘**：用药提醒触发 → 360° 舵机 + PillBoxTurntable 自动顺时针转至目标药槽（双微动开关精确定位）
- **大字显示**：屏幕左侧 110×120 覆盖层，30px 大字显示剂量数字，14px 文字显示 "请取出" + "颗药 · 槽X"
- **确认归零**：用户按下 GPIO10 静音按键 → 停止语音 / 清除提醒 / 转盘自动归零至槽位 0（goHome）
- **OGG 语音**：提醒期间循环播放 med_reminder.ogg，GPIO10 按键停止循环并重置解码器

### 四、定时调度器（SchedulerServer）

> 📄 相关文档：[编译烧录与工作流程](build-and-workflow.md)

独立于用药提醒的通用定时调度系统：

- **Web 页面**：`http://<IP>/scheduler` 配置定时事件
- **时间校验**：云端时间同步后可用（year ≥ 2025），未同步时拒绝添加事件
- **事件类型**：LED 开/关（GPIO3），支持单次/每天重复/每天重复含截止日期
- **MCP 语音**：说"定时开灯" → LLM 调用 `add_schedule` 工具自动添加事件
- **实时检查**：esp_timer 每 2 秒轮询系统时间，匹配当前 HH:MM 即触发

### 五、健康监测（XiaoZhi + MAX30102）

> 📄 相关文档：[硬件方案 (MAX30102 引脚)](hardware.md) · [微信通知 (异常告警)](wechat-push.md)

1. 手指在位检测（信号强度阈值，10 秒超时）
2. 15-20 秒稳定数据采集
3. PPG 波形滤波 → 峰值识别 → 心率/血氧计算
4. 语音播报 + 屏幕大字显示结果
5. 异常数值温和提醒（心率 < 50 或 > 120，血氧 < 95%）
6. 异常数值微信告警

### 六、双通道配置（Web 页面 + 串口屏）

> 📄 相关文档：[数据存储与串口屏](data-and-screen.md)

药盒提供两种配置方式，数据通过 MedicineConfigServer 实时同步：

| 通道               | 适用场景                       | 硬件               | 交互方式           |
|--------------------|--------------------------------|--------------------|--------------------|
| **Web 配置页**     | 子女远程配置、初次设置         | 手机/PC 浏览器     | 触摸 + 键盘        |
| **淘晶驰串口屏**   | 老人本地操作、日常调整         | TJC 串口屏         | 触摸 + 大字体      |

- **Web 配置页**：`/medicine-config` 用药计划、`/scheduler` 定时调度、`/servo-debug` 舵机调试
- **串口屏** 提供简化的本地界面，面向老人：药品名称编辑、提醒时间调整、总开关、一键出药、健康检测入口
- 任一方修改后，另一方立即同步（详见上方"数据存储与多端联动"）
