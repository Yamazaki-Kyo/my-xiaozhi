- ## 检测心率血氧：
-心率血氧检测模块功能描述：

**硬件接入：**  
药盒通过 **I2C 总线** 外接 **MAX30102 心率血氧传感器模组**，由 ESP32-S3 作为主控直接驱动。MAX30102 负责采集指尖 PPG 原始数据，芯片内置 LED 驱动与光电接收阵列，本端通过 I2C 读取寄存器数据后，在 ESP32-S3 上运行心率（HR）与血氧饱和度（SpO₂）解算算法（如基于峰谷检测或开源算法库），得出最终生理参数。


**语音触发与模式切换：**  
用户在药盒处于**小智已唤醒态**时，可通过自然语音指令进入检测模式，支持的触发话术包括但不限于：**“检测心率”**、**“测血氧”**、**“打开健康检测”**、**“我要测心率”**。小智本地 ASR 识别到上述意图后，向检测模块发送启动信号，系统切换至**健康检测模式**。

进入检测模式时，小智首先语音播报：**“请把手指轻轻放在传感器上，保持不动，即将开始检测。”** 同时 LCD 屏幕切换至检测专属界面，提示用户放置手指。


**检测流程：**  
1. **手指在位检测**：模块持续读取 MAX30102 的 IR/RED 原始数据，判断是否有有效手指按压（信号强度阈值过滤）。若 10 秒内未检测到有手指，语音播报：**“未检测到手指，请重新放置。”** 并重新开始计时；若连续三次超时，退出检测模式并播报：**“检测已取消，如需检测请重新唤醒。”**  
2. **数据采集**：确认手指在位后，系统进行 **15~20 秒** 的稳定数据采集，期间 LED 指示灯常亮或呼吸闪烁表示正在工作。  
3. **算法解算**：采集完成后，对 PPG 波形进行滤波、峰值识别与 SpO₂ 比例计算，输出最终心率值（单位：次/分钟）与血氧值（单位：%）。  
4. **结果播报**：小智语音播报检测结果，例如：**“检测完成，您的心率是每分钟 78 次，血氧饱和度 98%。”** 若数值超出常规安全范围（如心率 < 50 或 > 120，血氧 < 95%），播报末尾追加温和提醒：**“该数值偏离常规范围，如有不适请关注身体状况。”**

**LCD 显示反馈：**  
检测全过程中，LCD 屏幕实时展示检测状态，界面分为三个区域：
- **顶部状态栏**：显示“心率检测中”或图标动画；
- **中部实时区**：检测期间显示倒计时进度条（如“采集中… 15s”）；检测完成后切换为大字显示 **❤️ 心率：78 bpm** 与 **🫁 血氧：98%**，字体醒目，方便老年用户阅读；
- **底部提示区**：根据阶段显示“请按住传感器”、“保持稳定”、“检测完成”等引导文案。

**状态与并发控制：**  
- 健康检测模式为**独占模式**，检测期间转盘模块、服药提醒播报、出药动作等非紧急任务**全部挂起**，避免电机震动干扰 PPG 信号，也防止语音打断检测提示。  
- 检测完成后（无论成功、失败或用户主动取消），系统自动释放独占权，回到**闭口待机态**或恢复当日待执行的服药提醒队列。

**异常处理：**  
- **手指抖动/信号质量差**：若采集过程中信号波动过大导致算法无法收敛，语音提示：**“手指似乎有移动，请保持静止，重新检测。”** 并自动延长 5 秒采集时间，仅允许一次重试。  
- **传感器离线**：系统初始化时扫描 I2C 总线，若未检测到 MAX30102 设备，标记传感器故障，语音播报：**“健康检测功能暂时不可用。”** 此后语音指令触发检测时直接返回故障提示，不再尝试驱动硬件。

- ## 每日吃药输入：
-每日吃药计划配置与语音提醒模块功能描述：

**网页端 — 药品与服药计划管理**

药盒配套提供一个**移动端优先的响应式网页**（支持手机、平板、PC 访问），作为用户配置入口。网页采用**卡片式布局、柔和圆角、低饱和度配色**（如浅绿、米白、暖灰），避免医疗产品常见的冰冷感；字号适中，按钮热区充足，兼顾老年用户操作习惯。

网页内包含以下配置项：

1. **药槽绑定**  
   以可视化转盘示意图展示 7 个药槽（槽 1~7），用户点击某个槽位卡片后，进入该槽的药品编辑页。

2. **药品档案**  
   每个槽位可录入：药品名称（文本输入）、药品形状/颜色标签（可选图标，辅助记忆）、单次服用颗数（数字步进器，默认 1，上限 9）、每日服用次数（下拉选择 1~5 次）。

3. **提醒时间设置**  
   根据“每日次数”动态生成对应数量的时间选择器（例如每日 3 次则出现早/中/晚三个时间输入框，支持 24 小时制滚轮选择）。用户可为每次服药单独设定提醒时刻，也可一键套用常见模板（如“早 08:00、中 12:30、晚 19:00”）。

4. **计划总览与开关**  
   首页以时间轴或日历形式展示当日所有提醒节点；每个提醒卡片显示：时间、槽号、药品名、剂量。提供**总开关**控制提醒功能启停，也支持单个提醒的独立开启/关闭。

5. **数据持久化与同步**  
   网页配置完成后，用户点击“保存并下发”，药品计划通过 **HTTP/Wi-Fi** 推送至药盒主控，写入本地存储（如 ESP32 的 NVS 或外接 Flash）。药盒离线时网页可暂存云端或本地缓存，待设备上线后自动同步。

---

**设备端 — 提醒触发与语音播报**

药盒主控维护一份**今日服药计划表**，结构为按时间排序的提醒队列，每条记录包含：触发时间、目标槽号、药品名、单次颗数、是否已完成。

系统通过内部 RTC 或网络对时保持时钟准确。当系统时间**到达某条计划的触发时刻**，且该提醒处于开启状态、当日尚未执行过，则依次触发以下动作：

1. **小智语音播报**  
   调用本地语音合成模块（TTS）或预录音频，通过功放与喇叭播报提醒，例如：  
   *“主人，该吃药啦。请服用 1 号槽的降压药，每次 2 颗。”*  
   播报内容动态拼接药品名与剂量，确保用户明确知道该取哪个槽、取几颗。

2. **视觉/屏幕提示**  
   若药盒带屏幕，同步显示药品名称、槽号、剂量及药品图标；LED 可伴随温和呼吸灯闪烁，强化提醒而不刺耳。

3. **自动出药准备**  
   播报同时或播报结束后，转盘模块自动将对应药槽旋转至取药口（详见转盘模块描述），用户听到语音后可直接取药。

4. **取药确认与消提醒**  
   用户取药并按下确认键（或关闭盒盖）后，系统标记该次提醒为“已完成”，当日不再重复播报。若超时未确认（如 5 分钟），可进入温和二次提醒（降低音量重复播报一次），避免过度打扰。

**异常处理：**  
- 若药盒处于小智已唤醒态（用户正在交互），提醒播报礼貌插入，不中断当前对话。  
- 若当前转盘故障或该槽位未配置药品，播报内容自动调整为提示异常，避免空转。

- ## 药品转盘：
- 药盒内部设有一个水平旋转的药品转盘，由一枚 MG90S（180° 舵机） 通过中心转轴直接驱动。转盘上沿圆周均匀分布着 7 个储药槽（编号 1 至 7），每个槽在顶部开口，用于存放单次或单日的药量。此外，转盘在机械初始位置设有一个 闭口位置（编号 0），该位置没有开槽，是一个实心整体；当该位置对准药盒顶部的取药口时，取药口被实心面完全遮挡，从外面看药盒处于闭口状态，无法取药。
- 药盒外壳顶部只留有一个取药口，机械结构保证 任意时刻有且仅有一个位置（要么是闭口，要么是某一个药槽）对准取药口并暴露在外，其余位置均被外壳遮挡。
- MG90S 的 180° 行程被均分为 8 个位置，相邻位置间隔 22.5°，具体角度映射如下：
- 位置 0（闭口/待机）：0°
- 位置 1（药槽 1）：22.5°
- 位置 2（药槽 2）：45°
- 位置 3（药槽 3）：67.5°
- 位置 4（药槽 4）：90°
- 位置 5（药槽 5）：112.5°
- 位置 6（药槽 6）：135°
- 位置 7（药槽 7）：157.5°
- 系统上电或处于日常待机状态时，舵机默认停在 位置 0（闭口），此时用户看到的是被实心面遮挡的取药口，表示当前非服药时间。
- 当到达预设的服药时间，系统根据当日用药计划查询出本次应服用的药品所对应的槽号（1~7），驱动舵机从当前位置旋转至目标药槽位置，使该药槽对准取药口，用户即可取药。转动完成后，系统进入等待取药状态。
- 用户取药完成后（通过按键确认或盒盖关闭信号触发），舵机自动转回 位置 0（闭口），恢复待机遮挡状态。
- 模块需记录当前所处位置（0~7），转动期间不响应新的出药指令；若转动超时未到达目标位置，视为机械故障并停止，进入错误状态。
